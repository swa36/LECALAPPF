# ТЗ на API обмена сайта с 1С

Документ описывает, какие данные сайт получает из 1С и какие данные сайт отправляет в 1С.

Основано на текущем коде:

- `src/lekala_class/class_1C/ExChange1C.py`
- `src/lekala_class/class_1C/GetData1C.py`
- `src/lekala_class/class_1C/ExchangeOrder1CtoMarket.py`
- `catalog/tasks.py`
- `order/tasks.py`

## 1. Общие требования

### 1.1. Формат обмена

Обмен выполняется через HTTP API 1С.

Текущий формат:

```text
JSON
OData
application/json;odata=nometadata
```

К каждому `GET`-запросу сайт добавляет параметр:

```text
$format=application/json;odata=nometadata
```

### 1.2. Авторизация

Используется Basic Auth.

```text
LOGIN_1C
PASSWORD_1C
```

### 1.3. Базовый URL

```text
BASE_URL_1C
```

Все endpoint ниже указываются относительно `BASE_URL_1C`.

### 1.4. Направления обмена

```text
GET  - сайт получает данные из 1С
POST - сайт отправляет данные в 1С
```

## 2. GET: что сайт получает из 1С

## 2.1. Товары / номенклатура

### Назначение

Сайт получает список товаров из 1С и создает/обновляет товары в базе сайта.

### Endpoint

```http
GET Catalog_Номенклатура
```

### Фильтр

```text
IsFolder eq false and DeletionMark eq false
```

### Поля, которые сайт получает из 1С

| Поле 1С | Что означает | Куда идет на сайте |
|---|---|---|
| `Ref_Key` | UUID товара в 1С | `Product.uuid_1C` |
| `Parent_Key` | UUID категории товара | `Product.category` |
| `DataVersion` | Версия изменения объекта 1С | `Product.data_version` |
| `IsFolder` | Признак папки | Используется только в фильтре |
| `Code` | Код товара в 1С | `Product.code_1C` |
| `Артикул` | Артикул товара | `Product.article_1C` |
| `ВесЧислитель` | Вес | Сейчас в коде запрашивается, но явно не сохраняется |
| `Description` | Наименование товара | `Product.name` |
| `Описание` | Описание товара | `Product.description` |
| `ФайлКартинки_Key` | UUID главного изображения | `Product.main_img_uuid` |
| `ДополнительныеРеквизиты` | Характеристики товара | `ValueAdditionalAttributes` |

### Ожидаемый ответ

```json
{
  "value": [
    {
      "Ref_Key": "uuid",
      "Parent_Key": "uuid",
      "DataVersion": "string",
      "IsFolder": false,
      "Code": "string",
      "Артикул": "string",
      "ВесЧислитель": 0,
      "Description": "string",
      "Описание": "string",
      "ФайлКартинки_Key": "uuid",
      "ДополнительныеРеквизиты": [
        {
          "Свойство_Key": "uuid",
          "Значение": "string"
        }
      ]
    }
  ]
}
```

## 2.2. Категории товаров

### Назначение

Сайт получает дерево категорий из 1С.

В 1С категории находятся в справочнике `Номенклатура`, где объект является папкой.

### Endpoint

```http
GET Catalog_Номенклатура
```

### Фильтр

```text
IsFolder eq true and DeletionMark eq false
```

### Поля, которые сайт получает из 1С

| Поле 1С | Что означает | Куда идет на сайте |
|---|---|---|
| `Ref_Key` | UUID категории в 1С | `Category.uuid_1C` |
| `Parent_Key` | UUID родительской категории | `Category.parent` |
| `Description` | Название категории | `Category.name` |

### Ожидаемый ответ

```json
{
  "value": [
    {
      "Ref_Key": "uuid",
      "Parent_Key": "uuid",
      "Description": "string"
    }
  ]
}
```

### Логика на сайте

Сайт строит дерево категорий по `Parent_Key`.

Корневая категория определяется так:

```text
Parent_Key = 00000000-0000-0000-0000-000000000000
```

Сейчас сайт обрабатывает до 4 уровней:

```text
main_cat
sub_cat
sub_sub_cat
sub_sub_sub_cat
```

## 2.5. Цены товаров

### Endpoint

```http
GET InformationRegister_ЦеныНоменклатуры_RecordType
```

### Поля, которые сайт получает из 1С

## 2.8. Состав заказа клиента

### Назначение

После получения заказа со статусом `ВПроцессеОтгрузки` сайт запрашивает сам документ заказа и получает состав товаров.

### Endpoint

```http
GET Document_ЗаказКлиента(guid'{Заказ}')
```

Где `{Заказ}` - UUID заказа из регистра `InformationRegister_СостоянияЗаказовКлиентов`.

### Поля, которые сайт получает из 1С

В ответе нужна табличная часть:

```text
Товары
```

Внутри `Товары` нужны поля:

| Поле 1С | Что означает | Куда идет на сайте |
|---|---|---|
| `Номенклатура_Key` | UUID товара в 1С | Ключ товара |
| `Количество` | Количество товара в заказе | Количество в резерве/отгрузке |

### Ожидаемый ответ

```json
{
  "Ref_Key": "uuid",
  "Товары": [
    {
      "Номенклатура_Key": "uuid",
      "Количество": 1
    }
  ]
}
```

## 3. POST: что сайт отправляет в 1С

## 3.1. Создание партнера

### Назначение

Сайт может создать партнера в 1С для заказа маркетплейса.

Важно: в текущей задаче отправки заказов `order/tasks.py` вызывается отправка заказа. Отдельный метод создания партнера есть в коде, но нужно подтвердить, когда он вызывается в бизнес-процессе.

### Endpoint

```http
POST Catalog_Партнеры
```

### Что сайт отправляет в 1С

| Поле 1С | Тип | Что отправляем |
|---|---|---|
| `НаименованиеПолное` | string | Имя покупателя или строка маркетплейса |
| `Description` | string | Имя покупателя или строка маркетплейса |
| `Клиент` | boolean | `true` |

### Пример тела запроса

```json
{
  "НаименованиеПолное": "OZON 123456789",
  "Description": "OZON 123456789",
  "Клиент": true
}
```

### Что сайт ожидает получить в ответ

```json
{
  "Ref_Key": "uuid"
}
```

`Ref_Key` сохраняется как UUID партнера.

После создания заказа сайт отправляет запрос на проведение документа в 1С.

### Endpoint

```http
POST Document_ЗаказКлиента(guid'{Ref_Key}')/Post()
```

Где `{Ref_Key}` - UUID созданного документа `ЗаказКлиента`.

### Что сайт отправляет в 1С

Пустой `POST`.

```text
Тело запроса пустое
```

### Что сайт ожидает получить в ответ

Успешный HTTP-ответ от 1С.

Ожидаемо:

```text
200 OK
или
204 No Content
```

## 5. Порядок обмена

## 5.1. Обновление каталога на сайте

Сайт выполняет:

1. Получить товары: `GET Catalog_Номенклатура`, где `IsFolder = false`.
2. Получить цены: `GET InformationRegister_ЦеныНоменклатуры_RecordType`.
3. Получить остатки: `GET AccumulationRegister_ЗапасыИПотребности_RecordType`.
4. Получить названия характеристик: `GET ChartOfCharacteristicTypes_ДополнительныеРеквизитыИСведения`.
5. Получить виды цен: `GET Catalog_ВидыЦен`.
6. Получить категории: `GET Catalog_Номенклатура`, где `IsFolder = true`.
7. Создать/обновить товары на сайте.
8. Для новых или измененных товаров получить изображения из 1С.
9. После обновления сайта отправить обновленные остатки в маркетплейсы.

## 5.2. Отправка заказа с сайта в 1С

Сайт выполняет:

1. Находит заказы маркетплейсов, где `exchange_1c = false`.
2. Формирует тело документа `ЗаказКлиента`.
3. Отправляет `POST Document_ЗаказКлиента`.
4. Получает `Ref_Key` созданного заказа.
5. Отправляет `POST Document_ЗаказКлиента(guid'{Ref_Key}')/Post()`.
6. Помечает заказ на сайте как переданный в 1С.
